   - Расширяемый интерфейс для разных типов уровней

4. **LevelType.java** (8 строк):

5. **LevelDifficulty.java** (9 строк):

6. **CoreLevel.java** (16 строк):

7. **DronesLevel.java** (16 строк):

### ✅ Service Layer (Сервисный слой)

1. **LevelFactory.java** (12 строк):
   - Интерфейс фабрики уровней
   - Метод: `createLevel(String id)`
   - Паттерн Factory для создания уровней

2. **DefaultLevelFactory.java** (38 строк):
   - Реализация фабрики уровней
   - Spring Component (@Component)
   - Switch-expression для создания уровней по ID
   - Регистрация всех уровней:
     - 1.1, 1.2, 1.3, 1.boss (DronesLevel)
     - 2.1, 2.2, 2.3, 2.boss (SecurityLevel)
     - 3.1, 3.2, 3.3, 3.boss (CoreLevel)
   - Обработка неизвестных уровней (IllegalArgumentException)

3. **LevelCatalog.java** (55 строк):
   - Реализация каталога уровней
   - Spring Component (@Component)
   - Использует LinkedHashMap для хранения уровней
   - Инициализация через LevelFactory в конструкторе
   - Регистрация всех 12 уровней
   - Методы: getAllLevels(), getById(), exists()

4. **LevelCatalogService.java** (18 строк):
   - Интерфейс-контракт каталога уровней
   - Методы: getAllLevels(), getById(), exists()

5. **PlayerProfileService.java** (65 строк):
   - Сервис работы с профилем игрока
   - Spring Service (@Service)
   - Использует LevelCatalogService и InMemoryPlayerProfileRepository
   - Метод initDemoProfile() - создает демо-профиль при старте
   - Методы:
     - getSingleProfile() - получение единственного профиля
     - updateProgress(String levelId) - обновление прогресса
     - updateNickname(String nickname) - обновление никнейма
   - Валидация существования уровня перед обновлением прогресса

6. **PlayerProfileServiceContract.java**:
   - Интерфейс-контракт сервиса профиля
   - Определяет методы для работы с профилем

7. **InMemoryPlayerProfileRepository.java** (45 строк):
   - In-memory реализация репозитория профилей
   - Spring Repository (@Repository)
   - Использует ConcurrentHashMap для хранения
   - Методы:
     - findSingleProfile() - поиск единственного профиля
     - save(PlayerProfile) - сохранение профиля
     - createNew(String nickname, String levelId) - создание нового профиля с UUID
   - Готовность к замене на БД в будущем

8. **PlayerProfileRepository.java**:
   - Интерфейс репозитория профилей
   - Определяет методы для работы с хранилищем

9. **ProgressionStrategy.java** (14 строк):
   - Интерфейс стратегии прогресса
   - Метод: getNextLevel(PlayerProfile, LevelCatalogService)
   - Паттерн Strategy для определения следующего уровня

10. **LinearProgressionStrategy.java** (35 строк):
    - Реализация линейной стратегии прогресса
    - Spring Component (@Component)
    - Определяет следующий уровень по orderIndex
    - Если профиль null или currentLevelId null - возвращает первый уровень
    - Ищет уровень с orderIndex > текущего
    - Возвращает null если следующий уровень не найден

### ✅ API Layer (API слой)

1. **GameController.java** (71 строка):
   - REST контроллер для игрового API
   - Spring RestController (@RestController)
   - Базовый путь: "/api"
   - CORS: разрешен доступ с http://localhost:3000
   - Зависимости: LevelCatalogService, PlayerProfileServiceContract, ProgressionStrategy
   - Эндпоинты:
     - GET /api/levels - получение списка всех уровней
     - GET /api/profile - получение профиля игрока
     - POST /api/profile/progress - обновление прогресса (текущий уровень)
     - POST /api/profile/nickname - обновление никнейма
     - GET /api/profile/next-level - получение следующего уровня
   - Использует DTO для передачи данных
   - Обработка ошибок через ResponseEntity

2. **PlayerProfileDto.java** (54 строки):
   - DTO для передачи профиля на фронтенд
   - Поля: id, nickname, currentLevelId
   - Статический метод fromDomain() для конвертации из доменной модели
   - Геттеры и сеттеры
   - Конструкторы (пустой и с параметрами)

3. **UpdateProgressRequest.java**:
   - DTO запроса для обновления прогресса
   - Поле: levelId
   - Валидация через Jakarta Validation

4. **UpdateNicknameRequest.java**:
   - DTO запроса для обновления никнейма
   - Поле: nickname
   - Валидация через Jakarta Validation

5. **ApiErrorResponse.java**:
   - DTO для ответов с ошибками
   - Поля: status, error, message, path

6. **GlobalExceptionHandler.java** (60 строк):
   - Глобальный обработчик исключений
   - Spring RestControllerAdvice (@RestControllerAdvice)
   - Обработка исключений:
     - IllegalArgumentException -> 400 Bad Request
     - MethodArgumentNotValidException -> 400 Validation Failed
     - Exception -> 500 Internal Server Error
   - Возвращает ApiErrorResponse с деталями ошибки
